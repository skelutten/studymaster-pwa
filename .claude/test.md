## 🧪 Testing & Reliability
- **Always create Vitest unit tests for new features** (components, functions, services, etc).
- **After updating any logic**, check whether existing unit tests need to be updated. If so, do it.
- **Tests should live alongside components** or in dedicated test directories.
  - Include at least:
    - 1 test for expected use
    - 1 edge case
    - 1 failure case
